-------------------------------- CORE SOCKET SETUP & CONFIGURATION --------------------------------

socket
- Returns: An integer (the new file descriptor) or -1 on error.
- Usage: Call this first to ask the OS for a new network endpoint (IPv4, TCP stream). It returns the ID number you will use for all future configuration.

	example:
		int setsockopt(int sockfd, int level, int optname, const void *optval, socklen_t optlen);

close
- Returns: 0 on success, or -1 on error.
- Usage: Pass it a file descriptor to destroy the endpoint and free up memory when you are done with a client or shutting down the server.

setsockopt
- Returns: 0 on success, or -1 on error.
- Usage: Pass it your socket ID and specific system flags to force a behavior. Primarily used with SO_REUSEADDR to allow instant port reuse after a server crash.

getsockname
- Returns: 0 on success, or -1 on error.
- Usage: Pass it your socket ID and an empty address structure. The OS fills the structure with the exact local IP and port the socket is currently bound to.

fcntl
- Returns: Varies based on command, always -1 on error.
- Usage: Pass it your socket ID and the O_NONBLOCK flag. Mandatory for a single-threaded server to ensure reading/accepting doesn't freeze the program.


-------------------------------- CONNECTION MANAGEMENT --------------------------------

bind
- Returns: 0 on success, or -1 on error.
- Usage: Pass it your master socket ID and a structure with your IP and port. This permanently locks the socket to that address so clients can find it.

listen
- Returns: 0 on success, or -1 on error.
- Usage: Pass it your bound master socket to flip it into "passive" mode, telling the OS to queue up incoming connections.

accept
- Returns: A new integer (a new file descriptor for the specific client) or -1 on error.
- Usage: Pass it your listening socket. It grabs the first client in the waiting queue and creates a dedicated connection just for them.

connect
- Returns: 0 on success, or -1 on error.
- Usage: Primarily for clients. Pass it a socket and the remote address of a server to initiate the TCP handshake.


-------------------------------- DATA TRANSMISSION --------------------------------

send
- Returns: Number of bytes successfully sent, or -1 on error.
- Usage: Pass it a connected client's socket ID, a buffer with your message, and the message length to push data to the network.

recv
- Returns: Number of bytes read, 0 if the client disconnected, or -1 on error.
- Usage: Pass it a client's socket ID and an empty local buffer to pull incoming network data into your program.


-------------------------------- RESOLUTION & IP FORMATTING --------------------------------

getprotobyname
- Returns: A pointer to a protocol structure, or NULL on error.
- Usage: Pass it a string (like "tcp") to look up the official system integer ID for that protocol.

gethostbyname
- Returns: A pointer to a hostent structure, or NULL on error.
- Usage: Legacy IPv4 function. Pass it a hostname (like "localhost") to resolve it into an IP address. Not thread-safe.

getaddrinfo
- Returns: 0 on success, or a non-zero error code.
- Usage: Modern, protocol-independent replacement. Resolves a hostname/port into a linked list of ready-to-use address structures.

freeaddrinfo
- Returns: void.
- Usage: Pass it the linked list generated by getaddrinfo to safely clean up the memory allocated by the OS.

inet_addr
- Returns: Network-formatted binary IPv4 address, or INADDR_NONE on error.
- Usage: Converts a human-readable IPv4 string (like "127.0.0.1") into binary format.

inet_ntoa
- Returns: A string pointer to a human-readable IP.
- Usage: Converts a binary IPv4 address back into a dotted-decimal string.

inet_ntop
- Returns: A string pointer to a human-readable IP, or NULL on error.
- Usage: Newer, safer string-conversion function that supports both IPv4 and IPv6.


-------------------------------- BYTE ORDERING (ENDIANNESS) --------------------------------

htons / htonl
- Returns: An integer (16-bit short / 32-bit long) in Network Byte Order.
- Usage: Host-To-Network. Use before plugging a local port (short) or IP (long) into a configuration structure.

ntohs / ntohl
- Returns: An integer (16-bit short / 32-bit long) in Host Byte Order.
- Usage: Network-To-Host. Use after reading a port or IP from the network to translate it back to your machine's local format.


-------------------------------- MULTIPLEXING (EVENT LOOP) --------------------------------

poll
- Returns: Number of file descriptors with activity, 0 on timeout, or -1 on error.
- Usage: Pass it your array of socket IDs. It puts the server to sleep until at least one socket is ready to read, write, or has an error.


-------------------------------- FILE METADATA & SEEKING --------------------------------

fstat
- Returns: 0 on success, or -1 on error.
- Usage: Pass it a file descriptor and an empty structure to retrieve metadata (size, permissions) about a file.

lseek
- Returns: The new offset position in bytes, or -1 on error.
- Usage: Moves the read/write pointer inside a regular disk file. (Cannot be used on network sockets).


-------------------------------- SIGNAL HANDLING --------------------------------

signal
- Returns: The previous signal handler function pointer, or SIG_ERR on error.
- Usage: Legacy method to intercept OS signals (like Ctrl+C) and route them to a custom function.

sigaction
- Returns: 0 on success, or -1 on error.
- Usage: Modern, robust method to intercept signals. Allows blocking other signals while your custom shutdown handler runs.

sigemptyset / sigfillset
- Returns: 0 on success, or -1 on error.
- Usage: Initializes a signal set to be completely empty or to contain all possible signals.

sigaddset / sigdelset
- Returns: 0 on success, or -1 on error.
- Usage: Adds or removes one specific signal from an existing signal set.

sigismember
- Returns: 1 if the signal is in the set, 0 if not, or -1 on error.
- Usage: Checks if a specific signal is currently part of a set.
