Core Socket Setup and Configuration
socket: Creates a new communication endpoint and returns a file descriptor (a unique integer) that you use to reference this specific socket in future operations.

close: Destroys the file descriptor. In networking, this terminates the connection and frees up the resources associated with the socket.

setsockopt: Configures specific behaviors on a socket. For example, it allows you to set timeouts, adjust buffer sizes, or permit a socket to bind to an address that is currently in a "time wait" state.

getsockname: Retrieves the local IP address and port number that the operating system has assigned to a specific socket.

fcntl: Stands for "file control." It modifies the properties of a file descriptor. In networking, it is most commonly used to change a socket from its default "blocking" mode (where it waits indefinitely for data) to "non-blocking" mode.

================================================================================

Connection Management
bind: Associates a socket with a specific local IP address and port. This is usually mandatory for servers so they have a known address to listen on.

listen: Marks a bound socket as "passive." This tells the operating system that this socket will be used to accept incoming connection requests rather than initiate them. It also sets the maximum length of the queue for pending connections.

accept: Pauses the server to wait for a client to connect. Once a client connects, it extracts the first request from the listening queue and creates a brand-new socket specifically for that individual client conversation.

connect: Used by a client to establish a connection to a specific remote server (IP and port).

Data Transmission
send: Transmits outgoing data over a connected socket to the remote peer.

recv: Reads incoming data from a connected socket into a local buffer.

================================================================================

Hostname and Address Resolution
getprotobyname: Translates a string representing a protocol (like "tcp" or "udp") into its corresponding official integer protocol number.

gethostbyname: Takes a human-readable hostname (like "example.com") and resolves it into an IP address. (Note: This is considered a legacy function because it primarily handles IPv4 and is not thread-safe).

getaddrinfo: The modern, protocol-independent replacement for gethostbyname. It takes a hostname and/or service (port) and returns a linked list of address structures that can be used directly with bind or connect, seamlessly handling both IPv4 and IPv6.

freeaddrinfo: Safely releases the memory that was dynamically allocated by the operating system when you called getaddrinfo.

================================================================================


IP Address Formatting
inet_addr: Converts an IPv4 address string (like "192.168.1.1") into a binary format that the network hardware understands.

inet_ntoa: Does the exact opposite of inet_addr. It takes a binary IPv4 address and converts it back into a human-readable dotted-decimal string.

inet_ntop: A newer, more robust string-conversion function that supports both IPv4 and IPv6 addresses. It converts "network to presentation" (binary to string).

================================================================================


Byte Ordering (Endianness)
Different computers store multi-byte numbers in different orders in memory (Little Endian vs. Big Endian). Networks demand a standardized order (Big Endian, known as "Network Byte Order"). These functions ensure data is read correctly regardless of the computer's architecture:

htons: Host to Network Short. Converts a 16-bit integer (usually a port number) from the computer's local format to network format.

htonl: Host to Network Long. Converts a 32-bit integer (usually an IPv4 address) to network format.

ntohs: Network to Host Short. Converts a 16-bit integer received from the network back to the computer's local format.

ntohl: Network to Host Long. Converts a 32-bit integer received from the network back to the local format.

================================================================================


Multiplexing (Handling Multiple Connections)
poll: Allows a program to monitor an array of file descriptors, waiting until one or more of them are ready for I/O (e.g., ready to read, ready to write, or experiencing an error). This is crucial for building servers that can handle many clients simultaneously without needing a separate thread for every single client. (Note: select is the older alternative).

================================================================================


File Metadata and Seeking
fstat: Retrieves detailed metadata about an open file descriptor (like a regular file on the disk), such as its size, permissions, and last modification time.

lseek: Moves the read/write pointer to a specific location inside a file descriptor. If you want to jump to the middle of a file before reading, you use this. (Note: This works on disk files, but cannot be used on sockets, pipes, or terminals).

================================================================================


Signal Handling
Operating systems use signals (like SIGINT when you press Ctrl+C) to interrupt a program.

signal: A basic, legacy way to tell the program, "When you receive this specific signal, run this specific function instead of crashing."

sigaction: The modern, robust way to handle signals. It allows you to examine or change a signal action, block other signals while your handler is running, and gather deep information about why the signal was sent.

Signal Set Operations:
When using sigaction, you often need to define groups (sets) of signals to block or unblock.

sigemptyset: Initializes a signal set to be completely empty.

sigfillset: Initializes a signal set to contain every possible signal.

sigaddset: Adds one specific signal to an existing set.

sigdelset: Removes one specific signal from an existing set.

sigismember: Checks if a specific signal is currently part of a set.
